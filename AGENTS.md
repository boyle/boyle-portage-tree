Gentoo overlay containing ebuilds for various tools.


## Build/Lint/Test Commands

### Running pkgcheck (QA)

```bash
# Scan entire overlay
PORTDIR_OVERLAY=/home/boyle/proj/portage pkgcheck scan

# Scan specific package
PORTDIR_OVERLAY=/home/boyle/proj/portage pkgcheck scan app-benchmarks/gputest
```

Run pkgcheck to check for QA issues.

### Building packages

```bash
export PORTDIR_OVERLAY=/home/boyle/proj/portage

# Generate manifest (checksums)
ebuild ./pkg-1.0.ebuild manifest

# Full build cycle
ebuild ./pkg-1.0.ebuild clean unpack compile install

# Build from clean (removes work dir first)
ebuild ./pkg-1.0.ebuild clean && ebuild ./pkg-1.0.ebuild unpack prepare compile

# Individual phases
ebuild ./pkg-1.0.ebuild unpack   # Extract source
ebuild ./pkg-1.0.ebuild prepare  # Apply patches, run src_prepare
ebuild ./pkg-1.0.ebuild compile  # Build (usually no-op for binary)
ebuild ./pkg-1.0.ebuild install  # Install to staging dir
```

## Code Style Guidelines

### Ebuild Structure

- **EAPI**: Use `EAPI=8` (current stable)
- **Quotes**: Always quote variables: `"${S}"` not `${S}`
- **Newline**: End file with trailing newline
- **Executable**: Never set executable bit on `files/*` helpers
- **$PV**: Do NOT set manually - derived from filename

### Variables

```bash
# Good
S="${WORKDIR}/package-${PV}"
DOCS="README.md CHANGELOG.md"

# Avoid
S=${WORKDIR}/package-${PV}   # Unquoted
```

### Python Packages

- Use `python-single-r1` for packages that need Python runtime
- Use `python-any-r1` for packages that only build with Python
- Set `PYTHON_COMPAT=( python3_{10..14} )` or current supported versions
- Add `${PYTHON_DEPS}` to RDEPEND for runtime deps

```bash
PYTHON_COMPAT=( python3_{10..14} )
inherit python-single-r1

DEPEND="..."
RDEPEND="... ${PYTHON_DEPS}"

REQUIRED_USE="${PYTHON_REQUIRED_USE}"
```

### Flutter Packages (flutter.eclass)

- Use `inherit flutter` for Flutter/Dart packages
- Set `FLUTTER_VERSION` to the Flutter SDK version used
- Set `FLUTTER_PUB_DEPS` with entries in `name@version` format (hosted) or `name@version@git+url@commit@path@ref` format (git)
- Run `./bin/flutter-update-pub-deps -i pkg.ebuild` to regenerate from upstream lockfile
- Run `ebuild pkg.ebuild manifest` after updating FLUTTER_PUB_DEPS
- pub.dev build suffixes (`+N`) in versions are significant — the lockfile references a specific upload, and the download URL must use the full version to get correct content
- After extraction, `flutter_src_unpack` rewrites `pubspec.yaml`'s `version:` line to include build suffixes — pub reads version from pubspec.yaml, not the directory name
- **SRC_URI prefix**: The eclass automatically adds `flutter-pub-` prefix to hosted dep filenames and `flutter-pub-git-` prefix to git dep archive filenames in SRC_URI to avoid collisions with other packages
- **Git deps → path deps**: The eclass automatically converts git dependencies to path dependencies during unpack. It downloads the GitHub archive at the specific commit, extracts the subpackage to the pub cache, and patches `pubspec.yaml` and `pubspec.lock` to use `source: path` instead of `source: git`. No network is needed during build.
- **GitHub archive URLs**: The eclass automatically generates GitHub archive URLs in SRC_URI from git dep entries. Only GitHub is supported; non-GitHub git deps are skipped with a warning.
- **No `src_fetch` phase**: The `src_fetch` function is not a standard EAPI 8 phase function. Network requirements in build phases should be handled via SRC_URI entries or `RESTRICT="network-sandbox"`.
- **--no-version-check**: Pass to every `flutter` invocation to prevent the Dart Flutter tool from running `git fetch --tags` (`FLUTTER_ALREADY_LOCKED=1` only prevents `cache.lock()`, not the version check).
- **FLUTTER_ALREADY_LOCKED=1**: Still set this env var to prevent Flutter's internal cache lock operations from trying to `git fetch`.
- **flutter_prepare_tools()**: Call in `src_prepare()` after `dart pub get` to create a minimal package_config.json for flutter_tools. This prevents PubDependencies from trying to resolve flutter_tools' 96+ pub.dev deps (which include `test:1.26.3`) at build time.
- **flutter_setup_plugin_symlinks()**: Call in `src_compile()` before `flutter build --no-pub` to create plugin symlinks normally generated by `regeneratePlatformSpecificTooling` during `pub get`.
- **--no-pub**: Use with `flutter build` to skip the internal `pub get`. Must be paired with manual plugin symlink setup (via `flutter_setup_plugin_symlinks`).

### Patches

- Place patches in `files/` directory
- Use `PATCHES` array to apply automatically:

```bash
PATCHES=( "${FILESDIR}/fix-build.patch" )
```

### Creating Patches

To create a clean patch that applies without fuzz:

1. Unpack the source:
   ```bash
   sudo rm -rf /var/tmp/portage/category/pkg-ver
   export PORTDIR_OVERLAY=/home/boyle/proj/portage
   ebuild ./category/pkg/pkg-ver.ebuild clean unpack
   ```

2. Copy the original file with `.orig` extension:
   ```bash
   cp path/to/file path/to/file.orig
   ```

3. Modify `path/to/file` with your intended fix (do NOT modify the `.orig` file)

4. Generate the diff:
   ```bash
   diff -u path/to/file.orig path/to/file
   ```

5. Edit the diff output to create the patch file:
   - Change `a/path/to/file` to `a/linux/` (or appropriate relative path)
   - Change `b/path/to/file` to `b/linux/`  
   - Or use `--strip=1` equivalent paths

6. Test the patch:
   ```bash
   mkdir -p /tmp/patch-test
   cp path/to/file.orig /tmp/patch-test/linux/my_application.cc
   cd /tmp/patch-test
   patch -p1 < /home/boyle/proj/portage/category/pkg/files/your-patch.patch
   # Verify only intended changes were applied
   diff /tmp/patch-test/linux/my_application.cc path/to/file
   ```

### RESTRICT Flags

- `RESTRICT="test"` - Package has no tests
- `RESTRICT="strip"` - Prebuilt binary (don't strip)
- `RESTRICT="primaryuri"` - Only fetch from SRC_URI (for .deb files)
- `RESTRICT="network-sandbox"` - Allow network access during build phases
- **No RESTRICT="network-sandbox" needed for Flutter**: Linux engine artifacts (`libflutter_linux_gtk.so`, `gen_snapshot`, `flutter_tester`, etc.) are bundled in the Flutter SDK archive (`flutter_linux_*-stable.tar.xz`) in `artifacts/engine/linux-x64*/`. No `precache` call is needed during build. Pass `--no-version-check` to every `flutter` invocation and set `FLUTTER_ALREADY_LOCKED=1` in `src_prepare()` to suppress the Flutter CLI's internal `git fetch --tags` version check.

### Common Issues to Avoid

1. **Missing Python eclass**: Don't use raw `dev-lang/python:3.x`, use eclass
2. **Unquoted variables**: Always quote `${S}`, `${FILESDIR}`, `${D}`
3. **Executable files**: `files/` helpers should not be executable
4. **Duplicate class definitions**: Check patched Python scripts for dupes
5. **Stale pub.dev distfiles**: When FLUTTER_PUB_DEPS changes (especially build suffixes like `+3`), delete old files from `/var/cache/distfiles/` and regenerate Manifest — Portage caches by filename, not URL
6. **pub.dev build suffixes**: Always use the full version including `+N` suffix in download URLs. The base URL returns the latest upload, which may differ from what the lockfile references.
7. **Content-hash warnings**: After patching pubspec.lock to convert git deps to path deps, pub will emit "content-hash doesn't match" warnings for many packages. This is expected and harmless — the lockfile was modified, and pub recalculates content hashes on next `pub get`.

### Category Selection

- Use existing Gentoo categories from `/var/db/repos/gentoo/profiles/categories`
- Common: `dev-lang`, `media-video`, `dev-util`, `sci-electronics`, `app-benchmarks`

## Known Issues

- **yosys**: gcc-15 warnings (`Wmaybe-uninitialized`, `Warray-bounds`) in upstream code.
- **clgpustress**: Hardcoded OpenCL include path in Makefile (`/home/mat/docs/...`).
- **gputest**: Patch warning "patch unexpectedly ends in middle of line" (minor fuzz, still applies).
- **goguma**: pubspec.lock may contain dependency overrides not reflected in pubspec.yaml (e.g. `web` version pinned to 1.x while transitive deps constrain 0.5.x). The lockfile is authoritative.
- **goguma**: Linux engine artifacts are bundled in the Flutter SDK archive — no `precache --linux` needed. The earlier emerge failure was caused by `FLUTTER_ALREADY_LOCKED` not being set, which allowed the Flutter CLI wrapper to attempt `git fetch --tags`.
- **goguma**: Flutter 3.38.x uses pub workspaces. `flutter_tools` depends on `test:1.26.3` (via workspace resolution), which is not bundled in the SDK. The `flutter_prepare_tools()` eclass function works around this by creating a minimal package_config.json for flutter_tools, preventing PubDependencies from trying to resolve it at build time.
- **flutter.eclass**: When FLUTTER_PUB_DEPS changes, the Manifest must be regenerated (`ebuild pkg.ebuild manifest`). If URLs change (build suffixes), delete stale distfiles from `/var/cache/distfiles/` first.
- **flutter.eclass**: The `flutter build --no-pub` flag skips both `pub get` AND `regeneratePlatformSpecificTooling`. Plugin symlinks (.plugin_symlinks/) must be created manually via `flutter_setup_plugin_symlinks()` before building.

## Utility Scripts

### check-updates

Checks for upstream updates of packages in the overlay.

```bash
./bin/check-updates                    # Run check
./bin/check-updates -p 4              # With 4 parallel requests
./bin/check-updates --clear-cache      # Clear API cache
```

Output format: `<icon> <package>: <upstream-version> [(installed <ver>)][(ebuild <ver>)]`

| Icon | Meaning | Action |
|------|---------|--------|
| (blank) | Up to date | None |
| `↑` | Update available | `emerge pkg` |
| `+` | Ebuild needs update | Create/update ebuild |
| `o` | Not installed | `emerge pkg` |
| `?` | Non-GitHub source | Manual check |
| `!` | API error | Investigate |
| `~` | Anomaly detected | Warning |

Example output:
```
  ollama:         0.18.2
+ obsidian:       1.12.4 (installed 1.6.7)
↑ verilator:      5.046 (installed 5.044)
? gputest:
! unknown-pkg:
~ broken-pkg:     2.0.0 (WARNING: ebuild 1.0.0 is behind installed)
```

### flutter-update-pub-deps

Regenerates FLUTTER_PUB_DEPS and checks FLUTTER_VERSION from upstream pubspec.lock.

```bash
./bin/flutter-update-pub-deps net-irc/goguma/goguma-0.9.1.ebuild   # Print to stdout
./bin/flutter-update-pub-deps -i net-irc/goguma/goguma-0.9.1.ebuild # Edit in-place
```
